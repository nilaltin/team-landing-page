# Team Landing Page – Planning Documentation

## Brainstormed Features and Elements
- Team landing page with a clear title/header
- Display all team members on one page
- Each team member shown with an avatar image and name
- Clickable avatars/names that link to individual profile pages
- Consistent layout and styling across the landing page
- Organized folder structure (profiles, images, css, docs)
- Basic responsive layout for different screen sizes
- Optional hover effects on team member cards
- Optional team description or class/project name

## User Stories
- As a visitor, I want to see all team members on one page so that I can easily identify the team.
- As a visitor, I want to click on a team member’s avatar or name so that I can view their individual profile.
- As a team member, I want my profile page to have its own CSS file so that I can customize my design independently.
- As a team member, I want a clear project structure so that collaboration and merging changes is easier.
- As a user on a smaller screen, I want the layout to adjust so that the page remains readable and usable.

## Minimum Viable Product (MVP)
The minimum set of features required for the project to function properly includes:
- A team landing page (`index.html`) that displays all team members
- Each team member shown with an avatar image and name
- Clickable links from the landing page to individual profile pages
- Individual profile pages named `[name]_profile.html`
- Each profile page includes its own avatar image and CSS file
- Basic styling applied through a shared CSS file for the landing page

## Feature Categorization

### Need to Have
- Team landing page with all members displayed
- Clickable avatars/names linking to profile pages
- Individual profile pages for each team member
- Correct file naming and folder structure
- Basic CSS styling for the landing page

### Nice to Have
- Responsive layout for mobile devices
- Hover effects on team member cards
- Team description or brief introduction section

### Won’t Have (Out of Scope)
- JavaScript functionality
- Forms, databases, or user authentication
- Advanced animations or frameworks (React, Bootstrap, etc.)

## Assumptions and Constraints

### Assumptions
- Each team member already has or can create an individual profile page
- Each team member can provide an avatar image
- The team landing page is informational only (no dynamic content)

### Constraints
- Project must be built using only HTML and CSS
- Team members must use Git and GitHub for collaboration
- Changes must be made using branches and pull requests
- Time and scope are limited to the requirements of the assignment
